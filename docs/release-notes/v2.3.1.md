# Bazarr+ v2.3.1

Bazarr+ v2.3.1 is a patch release for the v2.3 line. It tightens the new translation and provider-language flows from v2.3.0, fixes embedded-subtitle upgrade handling, and updates vulnerable runtime dependencies before the next stable build.

This release is based on `development` after v2.3.0.

---

## Security

- Updated `urllib3` from `2.6.3` to `2.7.0`.
  - Fixes GHSA-mf9v-mfxr-j63j, CVE-2026-44432: decompression-bomb safeguards bypassed in parts of the streaming API.
  - Fixes GHSA-qccp-gfcp-xxvc, CVE-2026-44431: sensitive headers forwarded across origins in proxied low-level redirects.
- Updated `dynaconf` from `3.2.12` to `3.2.13`.
  - Fixes GHSA-pxrr-hq57-q35p, CVE-2026-33154: remote code execution risk through insecure template evaluation in the `@jinja` resolver.
- Added a startup dependency regression test so the runtime requirements probe and `requirements.txt` stay aligned with the patched versions.

---

## Translation

- Added language-profile driven auto-translation with a visible **Translate From** path.
- Added translation metadata columns for source score, provider, and status.
- Fixed auto-translate media type handling so movie and episode flows match the correct job variant.
- Fixed double-translate guards for forced subtitles, hearing-impaired suffixes, and variant language suffixes.
- Treated missing subtitle history as already at threshold so auto-translate does not loop on incomplete history rows.

---

## Providers

- Added per-provider language exclusions so individual providers can be limited without changing the global language profile.
- Inverted the provider language-selection model to make exclusions easier to reason about.
- Kept Provider Hub and shipped-provider settings aligned with the provider-language changes.

---

## Upgrade Reliability

- Fixed embedded-subtitle history upgrade handling for existing databases with older history rows.
- Fixed the project site GitHub stars badge URL.
- Added pre-commit hooks for common lint, secret, and formatting checks.
